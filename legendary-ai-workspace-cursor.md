## Managing AI Context

In this secret mission, I will learn to control Cursor's context by. how Cursor indexes your codebase. Creating a .curserignore file to exclude context ans .cursorindexignore file for more flexible control over indexing.

![Image](http://learn.nextwork.org/serene_gray_noble_ruru/uploads/ai-workspace-cursor_s6q4upload)

### Performance and security benefits

Managing context improves performance by excluding unnecessary files, Cursor's AI can focus on the relevant parts of the project. And improves security by files allow you to keep sensitive information, such as API keys, personal data, or proprietary algorithms. These prevents the AI from inadvertently accessing or exposing confidential data.

### Comparing .cursorignore and .cursorindexingignore

.cursorignore file is like a "do not enter" sign for Cursor which any file or folder on list it will be conpletely invisible to Cursor. .cursorindexingignore files to skip during indexing, but still allow referencing.
